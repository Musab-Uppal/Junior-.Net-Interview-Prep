# Contributing to Junior .NET Developer Interview Prep

First off, thank you for considering contributing to this repository! It's people like you that make this a great resource for everyone preparing for their .NET interviews.

## 🤝 How Can I Contribute?

### 🐛 Reporting Bugs & Typos
If you find a typo, incorrect information, or broken links:
1. Check if an issue already exists.
2. If not, open a new issue using the **Bug Report** template.
3. Be specific about the file and the line where the issue occurs.

### ✨ Suggesting Enhancements & New Topics
If you think a topic is missing or an answer could be improved:
1. Open an issue using the **Feature Request** template.
2. Provide a brief explanation of why this topic is important for a junior .NET developer interview.

### 💻 Pull Requests
We love pull requests! If you're ready to fix a bug or add a new topic:
1. Fork the repo and create your branch from `main`.
2. Make your changes in your branch.
3. Ensure your formatting matches the rest of the repository (Markdown, clear headings, concise answers).
4. Issue a pull request using the provided **Pull Request Template**.

## 📝 Styleguides

### Markdown Guidelines
- Keep explanations clear, concise, and easy to understand for beginners.
- Use code blocks with `csharp` syntax highlighting for C# examples.
- Use tables where appropriate for comparisons (e.g., `AddTransient` vs `AddScoped`).
- Use emojis in headings to keep the content engaging, matching the current style.

### Code Examples
- Keep code examples minimal.
- Focus on the core concept being explained.
- Ensure the code follows standard C# naming conventions.

## 💬 Community

By participating in this project, you agree to abide by the [Code of Conduct](CODE_OF_CONDUCT.md). We expect all contributors to be respectful and constructive.

Thank you for contributing! 🚀
